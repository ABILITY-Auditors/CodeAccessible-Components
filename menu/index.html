Menu Example

Here is an example of a menu aria role using a roving focus focus management implementation.

Roving focus refers to how focus moves along with the widget element that is currently in focus. Focus is placed directly onto the subcomponent (such as menuitem). Focus is still managed using JavaScript, as opposed to placing each subcomponent in tabbing/focus order.

menus should be used sparingly. It is not meant for navigation or your average web experience. It is meant to present a menu similar to a desktop application, where each menu item is a specific action (read: not navigation) that can be enacted.

Typically, composite widgets such as the menu role are used because navigating through each subcomponent by tabbing would take a considerable amount of time and effort, and it would reduce the number of tab stops to navigate the entire web app.
